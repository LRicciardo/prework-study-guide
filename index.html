HTML

  • Acronym: HyperText Markup Language
  • File extension(s): .html or .htm
  • HTML gives the content structure.
  • There are 4 types of structural and data elements: Text, Image, Header and List
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • There are 3 types of content elements: Meta-related, Content division, and Semantic.
  • The meta-related elements defines metadata such as the character set, description, keywords, author, and viewport.

CSS

  • Acronym: Cascading Style Sheets
  • File extension(s): .css
  • CSS gives the content style.
  • CSS can be inline. The style will only apply to that gives the content style.
  • CSS creates and formats the style rules for specific elements and style options
  • --- by defining the colors, aesthetics, visual layout, and organization of the webpage.
  • CSS can be used internal to the HTML or using an external style sheet.
  • Inline CSS is and internal use of CSS and can override an external CSS and only applies to that tag.
  • An internal style sheet uses CSS rule; is contained in the head section; and applies only to that webpage.
  • --- Also this type of style sheet will override an external CSS.
  • An external CSS can set the tone for all the webpages on the website. Changes can be made in one location.
  • Use link tag in the head section.
  • CSS rule has a selector and a {declaration block}.
  • Each declaration block has a property and a vlaue {prop1:val1; prop2:val2;}
  • A selector of * is a wildcard and applies to ALL elements.
  • Think about the CSS box model...Margin/Border/Padding/Content
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • side note: DRY => Do not Repeat Yourself - coding principle (like modular coding?)

Git

  • "Git is a version control system that lets you manage and keep track of each version of your project."
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories.
  • Each project/directory has 3 states:
  • ---The .git directory is where Git stores the metadata and object database for repo.
  • ---The working directory contains a copy of one version of the git project.
  • ---The staging area/index file that stores info about what will be committed into the git repo
  • --- To create and switch to new feature branch
  • command: git status => shows current branch in repository
  • command: git checkout main => switches to main branch
  • command: git checkout -b branch-name => -b attribute creates a new branch and switches to new branch
  • --- To merge feature branch into main branch Finish Git Flow
  • command: git status => shows current branch in repository
  • command: git checkout branch-name => switches to feature branch
  • command: git status => verify current branch
  • command: git add -A => adds or includes files to current branch (-A adds all changes)
  • command: git commit -m "text" => creates versions that need to be merged with message ??
  • command: git pull origin main => update to main branch before opening PR
  • command: git push origin branch-name => send local changes to feature branch
  • --------go to GitHub to compare/pull request
  • -----------resolve issue
  • --- Update local repo with update/merged feature
  • command: git checkout main => switches to main branch
  • command: git status => verify current branch
  • command: git pull origin main => update to main branch
  • --------go to GitHub to compare/pull request
  • -----------resolve issue
  • --- config: To setup/update personal info associated with the GitHub
  • --- --- include --global attribute to affect all repos, omit it to affect only single current working repo
  • command: git config --global user.name "your name" => changes name
  • command: git config --global user.email "email@example.com" => changes commit email address
  • side note: command: code . => starts the Visual Studio (VS) Code editor

JavaScript

  • Abbreviation: JS
  • File extension(s): .js, .cjs, .mjs
  • JavaScript is a high-level programmin language.
  • Open browser console by right-clicking on the browser window and clicking Inspect.
  • ----- Then change to the Console tab.
  • I learned 2 new words: truthy and falsy.
  • -- truthy means is a boolean value of true.
  • -- falsy means 0, -0, on, "", null, undefined, NaN, or boolean value of false.
  • A JS array index starts at 0, so number of items in array is index +1
  • A function is predefined set of tasks. Functions can be user-defined or JS prepared functions.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
