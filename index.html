HTML

  • Acronym: HyperText Markup Language
  • File extension(s): .html or .htm
  • HTML gives the content structure.
  • There are 4 types of structural and data elements: Text, Image, Header and List
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • There are 3 types of content elements: Meta-related, Content division, and Semantic.
  • The meta-related elements defines metadata such as the character set, description, keywords, author, and viewport.

CSS

  • Acronym: Cascading Style Sheets
  • File extension(s): .css
  • CSS gives the content style.
  • CSS can be inline. The style will only apply to that gives the content style.
  • CSS creates and formats the style rules for specific elements and style options
  • --- by defining the colors, aesthetics, visual layout, and organization of the webpage.
  • CSS can be used internal to the HTML or using an external style sheet.
  • Inline CSS is and internal use of CSS and can override an external CSS and only applies to that tag.
  • An internal style sheet uses CSS rule; is contained in the head section; and applies only to that webpage.
  • --- Also this type of style sheet will override an external CSS.
  • An external CSS can set the tone for all the webpages on the website. Changes can be made in one location.
  • Use link tag in the head section.
  • CSS rule has a selector and a {declaration block}.
  • Each declaration block has a property and a vlaue {prop1:val1; prop2:val2;}
  • A selector of * is a wildcard and applies to ALL elements.
  • Think about the CSS box model...Margin/Border/Padding/Content
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • side note: DRY => Do not Repeat Yourself - coding principle (like modular coding?)

Git

  • "Git is a version control system that lets you manage and keep track of each version of your project."
  • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories.
  • --- To create new feature branch
  • command: git status => shows files in working directory
  • command: git checkout branch-name => switches to that branch
  • command: git checkout -b branch-name => -b attribute creates a new branch and switches to new branch
  • --- To merge feature branch into main branch
  • command: git status => shows files in working directory
  • command: git add -A => adds or includes files to branch currently attached
  • command: git commit -m "text" => creates versions that need to be merged with message ??
  • command: git pull origin branch-name => recieve and merge any commits from the tracking remote branch
  • command: git push origin branch-name => sends local branch commits to remote repository branch
  • command: git checkout -b branch-name => creates a new branch and switches to new branch
  • command: git checkout branch-name => switches to that branch
  • side note: command: code . => starts the Visual Studio (VS) Code editor
